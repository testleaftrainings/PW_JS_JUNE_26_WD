File upload:
------------

Uploading files into a web application using Playwright.

types of upload:
---------------

1) Using <input type="file">  

The HTML tag name should be <input> and the attribute should be type="file".
Playwright directly injects/uploads the file using .setInputFiles().

syntax: await page.setInputFiles('input[type="file"]', 'path/to/file')

2) Using Event Listener

Used when the file upload button does not expose the input element directly.
Playwright listens for the file chooser event.

Event: filechooser
Method: .setFiles()

syntax: 

const fileChooserPromise = page.waitForEvent('filechooser');
await page.locator('#uploadBtn').click();
const fileChooser = await fileChooserPromise;
await fileChooser.setFiles('tests/data/sample.pdf');

File download
-------------

Playwright provides a download event to capture file downloads triggered from UI actions

Event: download
Method: .saveAs() -> used to save the downloaded file to a specific location.

syntax:

const downloadPromise = page.waitForEvent('download');
await page.locator('#downloadBtn').click();
const download = await downloadPromise;
await download.saveAs('downloads/sample.pdf');



In web automation or file systems, paths are used to locate a file or element. The two main types
are Absolute Path and Relative Path.

Relative Path
-------------
A relative path starts from the current location or a nearby element, not from the root.

Ex:
..\Documents\testdata.xlsx

Absolute Path
--------------

An absolute path gives the complete location starting from the root directory.

Ex:
C:\Users\Gauthami\Documents\testdata.xlsx


Note:
-----

await fDown.saveAs(path.join(__dirname, ../../Data, fDown.suggestedFilename()))
This line tells Playwright where to save the downloaded file.

>> path.join(__dirname, ../../Data, fDown.suggestedFilename())


🔹 1. path.join()
Comes from Node.js built-in path module.
It is used to safely combine multiple parts of a file path.
It automatically adds / or \ depending on your operating system (Windows, macOS, Linux).

Example:

path.join("folder", "subfolder", "file.txt")
Result (on Windows):
folder\subfolder\file.txt
Result (on Mac/Linux):
folder/subfolder/file.txt

So, path.join() helps build paths without worrying about slashes.

🔹 2. __dirname
It's a special variable in Node.js.
It gives the absolute path of the current JavaScript file's directory.

Example: If your file is located in:
C:\Projects\Playwright\Tests\DownloadTest.js

Then:
console.log(__dirname)
will print:

C:\Projects\Playwright\Tests


🔹 3. ../../Data
This part means:

Go two folders up from the current directory (../ twice).
Then go inside the “Data” folder.
So if your current file is:

C:\Projects\Playwright\Tests\DownloadTest.js

Then ../../Data points to:
C:\Projects\Data

🔹 4. fDown.suggestedFilename()

This gives the original name of the file being downloaded (like report.pdf, image.png, etc.).
Playwright reads this automatically from the browser's download info.

"join" is Putting it all together
path.join(__dirname, `../../Data`, fDown.suggestedFilename())
means:

Start from the folder where this test file is located (__dirname)
Move two levels up to reach the main project folder
Go inside the Data folder
Add the original file name automatically suggested by the browser.

Example:

C:\Projects\Data\MarPw.png

path.join(__dirname, '../../Data', fDown.suggestedFilename()) builds a full, correct path to save 
the downloaded file inside your Data folder, no matter what operating system or directory your test 
runs from.

The Playwright Config File is the central configuration file of a Playwright project
It defines how your tests should run, which browsers to use, timeouts, retries, reporters, 
screenshots, videos, traces, and many other settings.

testDir-> Specifies where test files are located.

timeout-> Maximum time allowed for a single test.

retries-> Number of times Playwright reruns a failed test

workers-> Controls parallel execution.

fullyParallel->Run tests in parallel.

forbidOnly->Prevents accidental commits of .only

process.env->A Node.js object containing environment variables. 
process.env.CI->Returns the value of the CI environment variable.

Local Machine:
--------------
process.env.CI = undefined (usually)
forbidOnly: !!process.env.CI
forbidOnly: false, Hence test.only() is allowed in local

CI Pipeline (Jenkins/GitHub Actions/Azure DevOps):
-------------------------------------------------
forbidOnly: !!process.env.CI
process.env.CI = "true"  //Error: test.only is not allowed and Fails execution in Pipeline
forbidOnly: true

! (Single NOT):Opposite boolean value
--------------------------------------
!true  -> false
!false -> true

!! (Double NOT):Converts any value into a boolean:
--------------------------------------------------

!!undefined -> false  //hence .only is allowed in local
!!0         -> false
!!""        -> false
!!"hello"   -> true
!!1         -> true

reporter-> Generates test reporter

use Block-> Settings inside use apply to every test

ex: use: {
  headless: false, //headless: true(Browser runs in background.)
  screenshot: 'on',
  video: 'on',
  trace: 'on'  //Useful for debugging failures.
}

projects (multiple browsers)->Controls browser choice and execution configuration.

DOM Structure [Document object model]:
--------------
DOM is the browser's representation of a web page as a tree of elements that JavaScript and 
automation tools can access and manipulate.


<html>
  <head>
    <title>Testleaf Page</title>
  </head>
  <body>
    <div>
      <h1>Welcome to Testleaf</h1>

      <!-- Label and Textbox -->

</html>

CSS Selectors [Cascading style sheet]: 
-------------------------------------
CSS is used for style and design a web page
CSS selectors are fast because browsers understand them natively and can locate elements quickly.

1. By ID :
Syntax : #idValue 
Example : #username

2. By Class :
Syntax : .classValue
Example : .inputLogin

3. By Attribute :
[tagName[attribute='value']]
Example : input[id='username']

4. Descendant Combinator 
Example : form input

5. Child Combinator
Example : form > input

-------------------------------------------------------------------------------------

Attribute priority :
-----------------------
1️⃣ aria-label - Accessibility-based, stable, first choice
2️⃣ id - Unique, fast
3️⃣ name -Usually stable, not always unique
4️⃣ class - For styling, changes often → least reliable

Playwright locator methods :
----------------------------

nth(), first(), and last() are Playwright Locator API methods.

They help you choose which element to act on when a locator matches multiple elements.

Strict Mode Violation Error :
-----------------------------
When a locator matches more than one element, Playwright throws an error to prevent ambiguity.
To fix this, use nth(), first(), or last() to specify which element you want to interact with.

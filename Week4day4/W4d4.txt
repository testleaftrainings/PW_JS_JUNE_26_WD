Data Parameterization: 
---------------------

Feeding test data from external files instead of hardcoding values.
Useful for **data-driven testing**

Features:
---------

Avoid hard-coded test data
→ Test data is externalized, making tests flexible and reusable.

Test multiple scenarios using a single test
→ Same test logic can be executed with different data sets.

Improve test coverage
→ More inputs can be validated without writing new test cases.

Reduce duplicate test cases
→ One parameterized test replaces many similar tests.

Improve maintainability
→ Changes in data do not require code changes in test logic.


ways of implementation
----------------------
1)CSV :Handling .csv files for bulk data-driven tests.
2)JSON :Reading and processing JSON files for dynamic test data.
3)ENV :Utilizing .env files for environment-specific data.

Test Annotation(Hooks)
----------------------
Hooks are special methods in Playwright that execute before or after test execution to handle 
setup and cleanup activities, thereby controlling the execution flow.

Types of Hooks:
---------------

1) test.beforeAll
→ Executes once before all tests in a file
  Used for global setup (ex: data connectivity or test data setup)

2) test.beforeEach
→ Executes before every test
  Used for common pre-conditions (ex: login and navigation)

3) test modules

4) test.afterEach
→ Executes after every test
  Used for cleanup (ex: logout, clearing data, screenshots on failure)

4) test.afterAll
→ Executes once after all tests
  Used for global cleanup (ex: closing connections, removing test data)

| Hook           | When it runs          | Use Case                              |
| -------------- | --------------------- | ------------------------------------- |
| `beforeAll()`  | Once before all tests | Start server, create test data        |
| `beforeEach()` | Before every test     | Navigate to app, login                |
| `afterEach()`  | After every test      | Logout, take screenshot on failure    |
| `afterAll()`   | Once after all tests  | Delete test data, close DB connection |
